# Convex AI Agent System Documentation

## Overview

Your chatbot platform now uses **Convex's official AI Agent component** for dynamic, intelligent responses powered by real LLM models. All static responses have been eliminated in favor of true AI-powered conversations.

## Key Features

### ✅ Real AI Responses
- **OpenAI GPT-4o-mini** integration for fast, intelligent responses
- **No more static responses** - every reply is generated by AI based on instructions
- **Vector embeddings** for context retrieval and conversation history search

### ✅ Document Integration
- **Automatic document search** during conversations
- **RAG (Retrieval Augmented Generation)** using uploaded PDF/document content
- **Context-aware responses** based on chatbot knowledge base

### ✅ Conversation Memory
- **Persistent chat threads** with full conversation history
- **Cross-conversation context** for better continuity
- **User-specific conversation tracking**

### ✅ Advanced Agent Features
- **Tool usage** for database queries and document search
- **Multi-step reasoning** for complex queries
- **Streaming responses** for real-time chat experience

## Usage Examples

### Basic Chatbot Response
```typescript
// Create a new conversation thread
const { threadId } = await ctx.runAction(api.ai.createChatbotThread, {
  chatbotId: "chatbot_id",
  userId: "user_123",
  title: "Customer Support Chat"
});

// Send message and get AI response
const response = await ctx.runAction(api.ai.sendMessageToAgent, {
  threadId,
  prompt: "How can I reset my password?",
  chatbotId: "chatbot_id",
  userId: "user_123"
});

console.log(response.text); // AI-generated response based on instructions
```

### Document-Aware Conversation
```typescript
// The AI automatically searches relevant documents
const response = await ctx.runAction(api.ai.sendMessageToAgent, {
  threadId,
  prompt: "What are your pricing plans?",
  chatbotId: "chatbot_id"
});

// Response will include information from uploaded pricing documents
// The AI uses the searchChatbotDocuments tool automatically
```

### Streaming Responses
```typescript
const streamResponse = await ctx.runAction(api.ai.streamChatbotResponse, {
  threadId,
  prompt: "Tell me about your product features",
  chatbotId: "chatbot_id"
});

console.log(streamResponse.text); // Real-time AI response
```

## Advanced Features

### Context Options
Configure how much conversation history to include:

```typescript
contextOptions: {
  includeToolCalls: false,     // Hide tool usage from conversation
  recentMessages: 10,          // Number of recent messages
  searchOptions: {
    limit: 5,                  // Max search results
    textSearch: true,          // Enable text search
    vectorSearch: true,        // Enable vector search
    messageRange: { before: 1, after: 1 }
  },
  searchOtherThreads: false    // Search across user's other chats
}
```


## Troubleshooting

### Common Issues

1. **"Chatbot not found" error**
   - Verify chatbot ID exists in database
   - Check chatbot `isActive` status

2. **OpenAI API errors**
   - Verify `OPENAI_API_KEY` environment variable
   - Check API key permissions and billing

3. **Document search not working**
   - Ensure documents have `status: "processed"`
   - Verify document content is not empty
   - Check `by_chatbot` index exists

4. **Thread not found**
   - Verify thread was created successfully
   - Check thread ID format (must be valid Convex ID)

## Best Practices

### Instruction Writing
- **Be specific**: Clear, detailed instructions work better
- **Include examples**: Show the AI how to respond
- **Set boundaries**: Define what the AI should/shouldn't do
- **Personality**: Define tone, style, and character

### Document Preparation
- **Clean content**: Remove formatting artifacts
- **Structured data**: Use headings and clear sections
- **Relevant info**: Focus on customer-facing content
- **Regular updates**: Keep knowledge base current

### Conversation Design
- **Thread persistence**: Use consistent thread IDs
- **User identification**: Track users for personalization
- **Context limits**: Don't overload with too much history
- **Error handling**: Graceful fallbacks for edge cases
